《精通Git》读书笔记 之 Git基础

2018-12-16 • 标签: Git

一. 概述

####

版本控制系统比较

  • Git：将数据存储为随时间变化的项目快照（将数据作为快照流）；提交/保存项目时，抓取所有文件当前状态的快照，存储指向该快照的引用。
  • CVS，Subversion等：存储对每个文件的基础版本所做的改动

特征

  1. 几乎所有操作都在本地执行
  2. 完整性：所有数据都会在存储前做校验和计算，以校验和引用对应数据（SHA-1散列机制）
  3. 通常只增加数据

文件三种状态

  • 已提交（committed）：存入本地数据库
  • 已修改（modified）：改动，但未存入本地数据库
  • 已暂存（staged）：标识了已修改文件的当前版本，加入了下一次要提交的快照中

三个区域

  • Git目录（仓库）：保存项目元数据和对象数据库
  • 工作目录：项目某个版本单次检出，Git目录 -> 磁盘
  • 暂存区：下次要提交的内容

Git基本工作流

  1. 修改工作目录中的文件
  2. 暂存文件，文件快照加入暂存区
  3. 提交暂存区文件，永久保存在Git目录

二. Git基础操作

####

获取Git仓库

  1. 在现有目录初始化Git仓库：git init
  2. 克隆现有仓库：git clone

在Git仓库记录变更

工作目录中文件两种状态：
已跟踪（上次快照中包含的文件：未修改，已修改，已暂存）；
未跟踪（既不在上次快照中，也不在暂存区中的文件）

  1. 查看当前文件状态：git status（简洁版：git status -s）
  2. git add：添加内容到下一次提交中
  3. 忽略文件：.gitignore

  4. 查看已暂存和未暂存的变更：git diff

    查看尚未添加到暂存区的变更：git diff
    查看会进入下次提交的已暂存的内容：git diff –staged

  5. 提交变更：git commit
